# Non-Functional Requirements

## Product Requirements

| Sr.# | Requirement/Description |
|------|------------------------|
| NFR-PR1 | The system must provide a user-friendly interface with minimal training required, including clear button/function layout, readable screen, and intuitive design |
| NFR-PR2 | The system must maintain fast transaction processing times to avoid delays during heavy traffic. |
| NFR-PR3 | Must support role-based access control, ensuring employees, managers and admin have access only to specific relevant features. |
| NFR-PR4 | The system must be compatible with standard POS hardware such as barcodes, printers and cash registers. |

## Organizational Requirements

| Sr.# | Requirement/Description |
|------|------------------------|
| NFR-OR1 | The system must allow customization according to the type of business, enabling customer features such as rewards |
| NFR-OR2 | Must include a reporting system for business insight (ie. sales reports, expenses, profit analysis) |
| NFR-OR3 | Must be able to integrate employee portal to access payroll and/or schedules |
| NFR-OR4 | The system must store and secure sensitive data such as customer information in compliance with privacy standards and laws |

## External Requirements

| Sr.# | Requirement/Description |
|------|------------------------|
| NFR-ER1 | The system must run on a commonly available OS (e.g Windows, macOS) to support a range of business owners from new and old. |
| NFR-ER2 | The system must function in an offline mode to access basic operations, with availability to sync data once online. |
| NFR-ER3 | The system must support regularly scheduled or emergency updates and maintenance, ensuring long term reliability. |
| NFR-ER4 | Must provide basic level customer support for troubleshooting and onboarding processes. |
