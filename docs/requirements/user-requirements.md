# User Requirements

| Sr.# | Requirement/Description | Owner/Stakeholder | Traceable System Requirements |
|------|------------------------|-------------------|-------------------------------|
| U1 | The system must allow the store manager to track inventory levels for all products, input orders, and ability to alter current inventory | Store Manager | S2 |
| U2 | The system must be able to let the management know when inventory stock is low through reminders or notification alerts | Store Manager | S2 |
| U3 | The system must allow staff to clock in/out, track hours worked, allow manager access to hours with ability to alter if needed. | Store Manager, Staff | S10 |
| U4 | The system must allow staff to process customer orders and accept multiple payment types efficiently. | Staff, Customer | S3 |
| U5 | The system must allow creation and modification of employee schedules with ability to export in different formats or share to staff, staff being able to view the schedule once completed. | Store Manager, Staff | S10 |
| U6 | The system must allow sales staff to process and view customer loyalty points during transactions | Staff, Customer | S1, S3 |
| U7 | The system must allow the IT support team to maintain and troubleshoot system issues. | IT support team | S9 |
| U8 | The system must allow kitchen staff to view the current inventory level to plan for service and food preparation. | Staff | S2 |
| U9 | The system must allow the owner to view daily sales total and transaction summaries. | Owner | S7, S10 |
| U10 | The system must allow the store manager or owner to add new menu items with descriptions, images, prices, and categories | Owner, Manager | S5 |
| U11 | The system must allow kitchen staff to view current menu items and their preparation requirements and steps left from management or previous staff. | Staff | S5 |
| U12 | The system must allow the owner to calculate profit margins by comparing revenue to expenses | Owner | S7, S10 |
| U13 | The system must allow the owner to export financial reports for tax preparation and accounting | Owner | S7 |
| U14 | The system must allow the owner to identify popular items on the menu, unpopular items, and peak sales periods | Owner | S5, S7 |
| U15 | The system must allow multiple staff to log into separate POS terminals with employee ID and password for security | Staff, Manager | S1, S4, S8 |
| U16 | The system must support different levels of access between staff and owner, with store management having access to most features and keeping sales and kitchen staff to limited, and owner full access. | Store Manager, Owner | S1, S4, S8 |
| U17 | The system must allow upper management to deactivate/activate employee accounts, and monitor account activity and system usage. | Store Manager, Owner | S1, S4 |
| U18 | The system must allow the creation of customer loyalty accounts done by sales staff on a POS system that includes a point earning system and redeemable rewards set up by the store. | Customer, Staff | S1, S3 |
| U19 | The system must support need based reservation features which will be optional for restaurants or businesses that incorporate the system into their operation. | Owner | S6 |
